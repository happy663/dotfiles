# Issue最新コメント確認スキル

作業しているissueに追記された内容を確認し、調査ログとして出力しやすい形式で提示します。

## 実行方針

### 1. コンテキスト確認

まず、会話のコンテキストにissue内容が既に共有されているか確認します。

**Neovimから共有されている場合**:
- `octo://` バッファや直接共有されたissue内容がコンテキストに含まれている
- この場合、`gh issue view` を実行する必要はない
- 既存のコンテキストから直接内容を読み取る

**コンテキストにない場合**:
- `gh issue view [番号] --comments` でissue全体を取得
- 最新のコメントを確認

### 2. 出力形式

issueで調査ログを取っている場合を考慮した形式:
- マークダウン形式で整形
- コピー&ペーストしやすい構造
- 重要な情報を箇条書きで明示

## 出力例

```markdown
## 更新内容

[最新コメントまたは追記された内容]

### 関連情報

- [追加された情報やリンク]
- [参照URL]

### 次のアクション

- [必要に応じて、次に行うべきことを提案]
```

## 注意事項

- **重要**: コンテキストに既にissue内容がある場合、不要な`gh`コマンド実行を避ける
- 調査ログとして記録しやすいよう、構造化された形式で出力
- 必要に応じて、過去のコメントとの関連も示す
- Neovimのocto.nvimプラグインから共有されるissueバッファを優先的に使用
