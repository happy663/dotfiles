# 徹底的なコード解説

選択されたコードについて、以下の方針で徹底的に解説してください:

## 解説方針

### 1. 前提知識の確認
- ユーザーが基本的な概念を知らない前提で説明する
- 専門用語は必ず定義する
- 例え話を積極的に使用する

### 2. 段階的な説明構造
1. **全体の目的**: このコードは何をするためのものか
2. **構成要素の説明**: 主要な部分を分解して説明
3. **詳細解説**: 行単位で徹底的に解説
4. **動作フロー**: 実際にどう動くか

### 3. 説明の深さ
- 各行の意味を説明
- なぜそう書くのか（理由・背景）
- 他の書き方との比較
- 実際の動作例

### 4. 理解を助ける要素
- **例え話**: 日常生活に置き換えた説明
- **図解**: テキストベースの図で視覚化
- **対比**: 「〜ではなく〜」という説明
- **具体例**: 実際の値や動作例

### 5. 用語解説
専門用語が出たら即座に解説:
```
専門用語とは:
- 意味
- なぜ必要か
- 例え話
```

### 6. 注意点
- 上から目線にならない
- 「当然知っているはず」という前提を置かない
- 質問しやすい雰囲気を作る
- 理解度を確認する質問を最後に入れる

### 7. 構造化された出力
- 見出しを使って階層構造を明確に
- コードブロックで引用
- 重要な部分は太字やリストで強調

## 必須セクション

1. **前提知識**: このコードを理解するために必要な基礎概念
2. **全体構造**: コード全体の流れ
3. **行単位解説**: 各行の詳細な説明
4. **動作シナリオ**: 実際に動く時の流れ
5. **まとめ**: 重要なポイントの再確認
6. **確認質問**: 理解度チェック

必ず日本語で、丁寧に、段階的に説明してください。
